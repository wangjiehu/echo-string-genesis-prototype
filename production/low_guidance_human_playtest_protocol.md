# 低引导真人试玩协议

日期：2026-06-05

## 1. 目的

验证玩家在低引导模式下是否真正理解“扫描、取词、写词、组合反应”，而不是照着目标句或Boss配方执行。

自动回归、玩家路径审计和DOM点击回归只能证明规则可达；真人试玩才用于判断理解质量。

## 2. 试玩入口

本地入口：

```text
http://127.0.0.1:4174/index.html?mode=low
```

线上入口发布后可在URL后追加：

```text
?mode=low
```

## 3. 观察规则

- 不提前解释“正确路线”。
- 只允许回答界面操作问题，不解释谜题答案。
- 玩家卡住超过90秒时，先记录卡点，再允许读出当前结果反馈条。
- 不把自动回归的S级当成真人掌握。
- 只记录匿名编号，不记录姓名、联系方式、账号或其他身份信息。

## 4. 记录字段

建议直接使用[低引导真人试玩记录模板](./low_guidance_playtest_record_template.md)记录。

| 字段 | 记录方式 |
| --- | --- |
| 玩家编号 | P01、P02等匿名编号 |
| 设备 | 桌面 / 笔记本 / 手机 |
| 输入方式 | 鼠标 / 触控板 / 触屏 |
| 总用时 | 完成面板或观察者计时 |
| 每区用时 | 报告JSON或观察者记录 |
| 误操作 | 按事件日志和失败账本分类 |
| 检查点恢复 | 完成面板或报告JSON |
| Boss三层复述 | 玩家用自己的话解释三层反制 |
| Z5理解 | 玩家说明自己是在激发力场还是取词改写 |
| 首屏理解 | 玩家第一次点击前说出“我现在要做什么” |

## 5. 通过标准

| 项目 | 通过标准 |
| --- | --- |
| Z1 | 能在不看完整答案的情况下完成扫描、取词、写词、重击 |
| Z2 | 能解释轻盈和沉重的不同用途 |
| Z3 | 能理解破绽和结构词对伤害的影响 |
| Z4 | 能主动把温度和介质联系成组合反应 |
| Z5 | 不把力场柱当成必须剥离的词源 |
| Z6 | 能复述蒸汽、冻裂、力场三层的反制逻辑 |
| 完成后 | 能说出至少两个“世界状态被词改变”的例子 |

## 6. 失败判定

| 现象 | 结论 |
| --- | --- |
| 同一区卡住超过3分钟且无法说出下一步假设 | 提示强度不足 |
| 多次尝试剥离Z5力场柱 | Z5规则表达仍不统一 |
| Boss三层只按记忆配方执行，不能解释原因 | Boss渐进揭示不足 |
| 玩家不知道扫描前后差异 | 扫描价值仍不足 |
| 手机首屏找不到可点对象或扫描按钮 | 移动端信息层级失败 |

## 7. 试玩后处理

每轮试玩后只做三类修改：

1. 阻断主线或造成软锁的，立即修。
2. 三名以上玩家在同一区域同类误解，修提示或交互。
3. 单人偏好或审美反馈，先记录，不立即改规则。
