# 断句庭院谜题路线矩阵

用途：给关卡、程序和测试对齐每个谜题的主解法、旁路解法、失败反馈和防卡死方案。

## 1. 谜题总表

| ID | 区域 | 谜题 | 必需知识 | 主解法 | 旁路解法 | 防卡死 |
| --- | --- | --- | --- | --- | --- | --- |
| P01 | Z1 | 封文大门 | 扫描、剥离、注入 | 从裂纹石像取脆弱，注入大门后重击 | 从碎片取脆弱 | 碎片持续提供脆弱 |
| P02 | Z2 | 压机关石 | 质量槽替换 | 轻盈移动石头，沉重压住机关 | 用小石堆压机关，或搭斜坡越低墙 | 重置拉杆、浮字碑恢复 |
| P03 | Z4 | 冰封升降台 | 温度槽替换 | 过热融化冻结 | 用冻结水池制造临时冰面绕过半段 | 火盆周期恢复过热 |

## 2. 详细路线

### P01 封文大门

**设计意图**：最短路径验证核心交互。

**主路径**：

1. 扫描门。
2. 读到“封闭 / 坚固”。
3. 扫描裂纹石像。
4. 剥离“脆弱”。
5. 注入门。
6. 重击破门。

**测试观察点**：

- 玩家是否主动扫描石像。
- 玩家是否理解“脆弱”进入词库。
- 玩家是否知道注入后还需要物理打击。

**需要优化的风险**：

- 如果玩家以为注入脆弱后门会自动打开，破门反馈要加强，例如门先出现裂纹和重击提示。

### P02 压机关石

**设计意图**：避免词缀只像钥匙。质量词要有前后两种状态价值。

**主路径**：

1. 石头沉重，推不动。
2. 玩家从浮字碑获取轻盈。
3. 石头变轻，可推动。
4. 到机关板后，玩家需要恢复沉重。
5. 出口栅门打开。

**旁路限制**：

- 搭斜坡旁路只能越过低墙，不能跳过后续战斗触发。
- 小石堆压机关需要至少两个小石块，避免比主解法更简单。

**需要优化的风险**：

- 如果玩家忘记沉重来源，压机关石自身可在扫描时提示“原质量：沉重，可恢复”。

### P03 冰封升降台

**设计意图**：温度槽第一次用于探索。

**主路径**：

1. 升降台冻结无法启动。
2. 火盆提供过热。
3. 注入过热解除冻结。
4. 升降台启动。

**旁路限制**：

- 冻结水池制造冰面只能通往奖励平台，不能完全跳过升降台，否则玩家可能没学会融冰。

**需要优化的风险**：

- 过热的持续伤害不要惩罚玩家太重，第一次使用应偏安全。

### P04 蒸汽遮蔽

**设计意图**：让“潮湿”从弱状态变成反应钥匙。

**主路径**：

1. 玩家从水池获取潮湿。
2. 敌人或热石带有过热。
3. 注入潮湿触发蒸汽爆发。
4. 蒸汽遮蔽敌人视线，玩家通过或击败敌人。

**旁路限制**：

- 如果玩家不用蒸汽，也可以硬打，但时间更长且资源消耗更高。

**需要优化的风险**：

- 蒸汽云不能遮挡玩家自己的扫描UI。云层边缘需要留出清晰轮廓。

### P05 碎桥锚点

**设计意图**：验证力场词缀是否可读、可控。

**主路径**：

1. 吸引力场把碎片聚到桥中央。
2. 排斥力场把碎片推向锚点。
3. 三个锚点稳定后生成桥面。

**旁路限制**：

- 力场震荡会把碎片推得更远，适合熟练玩家，但失败后可重置。

**需要优化的风险**：

- 物理碎片不能太小。关键碎片必须有明显轮廓、发光边和吸附终点。

### P06 Boss护文层

**设计意图**：综合考核环境取词，而不是纯背词表。

**主路径**：

1. Boss结构槽锁定，直接注入脆弱失败。
2. 玩家通过环境反应打掉护文层。
3. 每次破绽后锁定削弱一层。
4. 第三次破绽后结构槽开放。
5. 注入脆弱并重击结束战斗。

**旁路限制**：

- 玩家可以用力场打断冲撞，但不能直接靠力场耗死Boss。

**需要优化的风险**：

- “结构槽锁定”必须解释清楚：不是游戏拒绝玩家，而是需要先破护文层。

## 3. 测试记录字段

每次试玩记录以下字段：

| 字段 | 说明 |
| --- | --- |
| `timeToFirstScan` | 从开始到第一次主动扫描 |
| `timeToFirstExtract` | 从开始到第一次剥离 |
| `timeToFirstInject` | 从开始到第一次注入 |
| `puzzleFailureCount` | 每个谜题失败次数 |
| `wrongTargetInjectCount` | 注入到错误对象的次数 |
| `reactionTriggered` | 是否触发过组合反应 |
| `bossSemanticActions` | Boss战中语义操作次数 |
| `softlockEvents` | 是否出现卡死或需要人工干预 |

## 4. 当前反思

P02和P05是最可能拖慢节奏的点。P02需要保证“恢复沉重”被玩家理解；P05需要防止物理碎片失控。首轮灰盒测试若节奏过慢，优先减少P05碎片和锚点数量，而不是砍掉Z4蒸汽反应，因为组合反应是MVP的关键验证项。
